9. 🔍 Auditoria de Código — admin-guard.js (FreeNameConvention)

Análise do módulo src/service/admin-guard.js (314 linhas) que controla toda a autenticação administrativa da aplicação.

Vulnerabilidade #1 — Rate Limiting In-Memory

// ⚠ _rate é uma variável em memória — reseta ao reiniciar a aplicação
const _rate = { count: 0, firstAt: 0, lockedUntil: 0 };
const MAX_TRIES = 5;
const LOCK_MS   = 5 * 60 * 1000;  // 5 min lockout
const WIN_MS    = 15 * 60 * 1000; // 15 min window

ALTO — Um atacante pode reiniciar a aplicação (ou esperar que ela reinicie) para resetar o rate-limiting e continuar o brute-force indefinidamente.

Recomendação: Persistir _rate no ConfigStore com timestamp. Implementar backoff exponencial.

Vulnerabilidade #2 — Legacy SHA-256 Ainda Aceito

function verifyPassword(password, storedHash) {
  if (!storedHash) return true;
  const clean = sanitize(password, 128);
  // ⚠ Legacy SHA-256 — sem salt, sem PBKDF2, vulnerável a rainbow tables
  if (!storedHash.startsWith('pbkdf2:')) {
    return crypto.createHash('sha256').update(clean, 'utf8').digest('hex') === storedHash;
  }
  // ✅ Caminho PBKDF2 moderno com timingSafeEqual
  ...
}

MÉDIO — Usuários que nunca reconfiguraram a senha ainda podem usar hash SHA-256 simples (sem salt). Vulnerável a ataques de dicionário e rainbow tables.

Recomendação: Forçar migração automática: se legacy SHA-256 for detectado e verificado com sucesso, re-hash imediatamente para PBKDF2.

Vulnerabilidade #3 — Senha Exposta em schtasks

// ⚠ Password visível na lista de processos (tasklist, Process Explorer)
cmd = `schtasks /create /f /tn "${task}" /tr "\\"${safe}\\" --service" 
       /sc ONLOGON /ru "${u}" /rp "${p}"`;

MÉDIO — A senha do usuário Windows é passada como argumento de linha de comando para schtasks, ficando visível em qualquer processo monitor.

Recomendação: Usar /rp * para prompt interativo, ou gravar via stdin pipe. Alternativamente, usar COM API do Task Scheduler.

14. 🔎 Consultas KQL Utilizadas

Queries executadas contra o workspace SentinelMaster (25a37558-19a8-4e28-b7ee-496d970317de).

Q1 — Falhas de login por tipo de erro

SigninLogs
| where TimeGenerated > ago(90d)
| where UserPrincipalName has "admin" or Identity has "admin"
| where ResultType != "0"
| summarize FailedAttempts=count(), DistinctIPs=dcount(IPAddress) 
  by UserPrincipalName, ResultType, ResultDescription, AppDisplayName
| order by FailedAttempts desc | take 50

Q2 — Sign-ins de alto risco (Identity Protection)

SigninLogs
| where TimeGenerated > ago(90d)
| where UserPrincipalName has "admin"
| where IsRisky == true or RiskLevelDuringSignIn in ("high", "medium")
| project TimeGenerated, UserPrincipalName, IPAddress, Location, 
  RiskLevelDuringSignIn, RiskDetail, AppDisplayName
| order by TimeGenerated desc | take 50

Q3 — Detecção de Brute-Force (>5 falhas/hora)

SigninLogs
| where TimeGenerated > ago(90d)
| where UserPrincipalName has "admin" and ResultType != "0"
| summarize AttemptCount=count() by IPAddress, Location, bin(TimeGenerated, 1h)
| where AttemptCount > 5
| order by AttemptCount desc | take 50

Q4 — Detalhes de falhas 50126 (credenciais inválidas)

SigninLogs
| where TimeGenerated > ago(90d) and UserPrincipalName has "admin"
| where ResultType == "50126"
| project TimeGenerated, UserPrincipalName, IPAddress, Location, 
  AppDisplayName, UserAgent, ConditionalAccessStatus
| order by TimeGenerated asc | take 50

Q5 — Logins bem-sucedidos por aplicativo

SigninLogs
| where TimeGenerated > ago(90d) and UserPrincipalName has "admin"
| where ResultType == "0"
| summarize SuccessCount=count(), DistinctIPs=dcount(IPAddress) 
  by UserPrincipalName, AppDisplayName, AuthenticationRequirement
| order by SuccessCount desc | take 30

Q6 — DeviceLogonEvents para contas admin

DeviceLogonEvents
| where TimeGenerated > ago(90d) and AccountName has "admin"
| where ActionType == "LogonFailed"
| project TimeGenerated, AccountName, DeviceName, RemoteIP, 
  RemoteDeviceName, FailureReason, LogonType, Protocol
| order by TimeGenerated desc | take 20
